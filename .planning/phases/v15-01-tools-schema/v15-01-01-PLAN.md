<objective>
将 tools.json 从自定义 params 格式迁移到 OpenAI 风格的 inputSchema (JSON Schema) 格式。

Purpose: 确保 LLM 可以正确理解工具参数结构，与 OpenAI Chat Completions API 的 tool calling 格式兼容。

Output: 迁移后的 tools.json 使用 inputSchema 格式
</objective>

<context>
@app/src/main/assets/tools.json (当前旧格式)

迁移前格式 (params):
{
  "tools": [
    {
      "name": "show_toast",
      "params": {
        "message": {
          "type": "string",
          "required": true,
          "description": "..."
        }
      }
    }
  ]
}

迁移后格式 (parameters):
{
  "tools": [
    {
      "name": "show_toast",
      "parameters": {
        "type": "object",
        "properties": {
          "message": {
            "type": "string",
            "description": "..."
          }
        },
        "required": ["message"]
      }
    }
  ]
}
</context>

<tasks>

<task type="auto">
  <name>Task 1: 更新 tools.json 为 inputSchema 格式</name>
  <files>app/src/main/assets/tools.json</files>
  <action>
将 tools.json 从旧格式迁移到新格式:

1. 将顶层 "params" 键重命名为 "parameters"
2. 在 parameters 内添加 "type": "object"
3. 将参数定义移入 "properties" 对象
4. 将每个参数的 "required": true/false 转换为 parameters.required 数组
5. 保留 "default" 字段在 properties 中
6. 保留 "description" 字段

示例转换:
旧: "params": { "message": { "type": "string", "required": true, "description": "..." } }
新: "parameters": { "type": "object", "properties": { "message": { "type": "string", "description": "..." } }, "required": ["message"] }
  </action>
  <verify>
cat app/src/main/assets/tools.json | python3 -c "import json,sys; d=json.load(sys.stdin); t=d['tools'][0]; assert 'parameters' in t; assert t['parameters']['type']=='object'; assert 'properties' in t['parameters']; assert 'required' in t['parameters']; assert 'message' in t['parameters']['required']; print('Format valid')"</verify>
  <done>tools.json 使用 inputSchema 格式: 包含 parameters.type=object, parameters.properties, parameters.required 数组</done>
</task>
